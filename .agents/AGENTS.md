# Agent Rules for Tello-VLM-project

- All python code must be written in a highly professional, modular, and reusable way.
- All code comments and docstrings must be written in English to ensure compatibility with standard developers' practices and AI analysis tools.
- Code should be heavily commented to explain engineering choices, edge cases, and optimization details.
- Provide step-by-step explanations and progress incrementally without overwhelming the codebase with too much code at once.
- Always explain the "why" behind specific choices (e.g. CUDA memory limits, queue sizes, OS limitations).
- Every function, method, and class MUST include detailed docstrings specifying:
  - The purpose of the function/class.
  - Parameters (including variable names, types, and descriptions).
  - Return values (types and descriptions).
  - Exceptions raised (if any).
